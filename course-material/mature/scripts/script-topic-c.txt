Folie 1 - for-Loops: Grundkonzept

Stell dir vor, du moechtest fuer jedes Element einer Sequenz, beispielsweise einer Liste dieselbe Aktion ausfuehren. Genau dafuer gibt es in Python die for-Loop - eine Form der Zaehlschleife.

Das Grundprinzip klingt denkbar einfach: FUER jedes Element IN einer Sequenz, fuehre eine Anweisungen aus.

Die Syntax in Python lautet: for, dann die Schleifenvariable, dann das Schluesselwort in, dann die Sequenz bzw. Liste - und alles, was in jedem Durchlauf ausgefuehrt werden soll, steht eingerueckt im Codeblock darunter.

Die Schleifenvariable kann beliebig benannt werden. Sie nimmt in jedem Durchlauf automatisch den Wert des naechsten Elements aus der Sequenz an. Der eingerueckte Codeblock kann beliebig viele Anweisungen enthalten.

---

Folie 2 - for-Loops: Beispiel (Einfuehrung)

Schauen wir uns ein konkretes Beispiel an. Wir haben eine Liste namens lectures, die drei Eintrage enthaelt: Maths, English und Computer Science.

Mit einer for-Loop koennen wir nun fuer jeden Eintrag automatisch eine Ausgabe erzeugen. Wir schreiben: for lecture in lectures, und im Codeblock darunter: Ausgabe des Satzes "I love" gefolgt vom aktuellen Wert der Variable lecture.

Was passiert nun Schritt fuer Schritt? Im ersten Durchlauf wird Maths aus der Liste geholt und der Variable lecture zugewiesen - ausgegeben wird "I love Maths". Im zweiten Durchlauf kommt English dran, dann Computer Science. Nach dem dritten Durchlauf ist die Liste erschoepft und die Schleife endet.

Das ist ein sehr wichtiges Konzept - merke dir dieses Muster gut, man verwendet es ständig.

Folie 3 - for-Loops: Beispiel (Struktur)

Schauen wir uns dasselbe Beispiel nochmal an, diesmal mit einem Blick auf die einzelnen Bestandteile des Codes.

Die erste Zeile definiert die Liste lectures - das ist die Sequenz, ueber die wir iterieren wollen.

Die zweite Zeile ist die for-Loop-Definition selbst: Sie legt fest, welche Variable in jedem Durchlauf das aktuelle Element aufnimmt - hier ist das lecture.

Der eingerueckte Codeblock darunter ist der wiederholte Teil: Er wird fuer jedes Element der Liste einmal ausgefuehrt.

Zusammengefasst: Die Sequenz liefert die Elemente, die Schleifenvariable traegt sie nacheinander weiter, und der Codeblock verarbeitet sie. Dieses Zusammenspiel ist das Herzstuck aller for-Loops.

Folie 4 - while-Loops: Grundkonzept

Neben der for-Loop gibt es in Python noch eine zweite wichtige Schleifenart: die while-Loop.

Der Unterschied liegt im Steuerungsprinzip. Eine for-Loop laeuft fuer jedes Element einer Sequenz. Eine while-Loop laeuft so lange, wie eine Bedingung erfuellt ist.

Das funktioniert so: SOLANGE die Bedingung gilt, fuehre die Anweisungen aus.

In Python lautet die Syntax: while, dann die Bedingung, Doppelpunkt, und darunter eingerueckt der Codeblock.

Ein einfaches Beispiel: x startet bei 0. Solange x kleiner als 3 ist, wird x ausgegeben und dann um 1 erhoeht. Die Schleife laeuft also fuer x gleich 0, 1 und 2 - sobald x den Wert 3 erreicht, ist die Bedingung nicht mehr erfuellt und die Schleife endet.

---

Folie 5 - while-Loops: Kombination mit User-Input

Ein besonders haeufiges Einsatzgebiet fuer while-Loops ist die Verarbeitung von Benutzereingaben bzw. User Input.

Stell dir vor, dein Programm soll so lange weiterlaufen, bis der Nutzer ein bestimmtes Wort eingibt. Das laesst sich elegant mit einer while-Loop umsetzen.

Im Beispiel: Eine Variable user_input wird auf einen leeren String gesetzt. Die while-Loop laeuft solange, wie user_input nicht gleich "quit" ist. Innerhalb der Schleife wird der Nutzer nach einer Eingabe gefragt, der eingegebene Text wird in user_input gespeichert und anschliessend ausgegeben. Sobald der Nutzer "quit" eingibt, ist die Bedingung nicht mehr erfuellt und die Schleife stoppt.

Alternativ kann man das Abbruchverhalten auch ueber eine sogenannte Flag steuern - das ist eine einfache Boolean-Variable, die auf True oder False gesetzt wird und so den Schleifenablauf kontrolliert.

---

Folie 6 - while-Loops: Endlosschleifen

Eine besondere Variante der while-Loop ist die Endlosschleife. Sie entsteht, wenn als Bedingung einfach True geschrieben wird - also eine Bedingung, die sich nie aendert.

Da True immer wahr ist, wuerde eine solche Schleife theoretisch unendlich lange laufen. In der Praxis bricht man sie gezielt mit dem Schluesselwort break ab.

Im Beispiel: Die Loop fragt in jedem Durchlauf nach einer Eingabe und gibt sie aus. Erst wenn der Nutzer "quit" eingibt, wird mit break die Schleife sofort beendet.

Dieses Muster ist in der Praxis sehr verbreitet - etwa bei Menueprogrammen oder Chat-Anwendungen, die kontinuierlich auf Eingaben warten sollen.

---

Folie 7 - Loops: Schluesselwoerter fuer den Ablauf

Es gibt drei Schluesselwoerter, die du im Zusammenhang mit Schleifen kennen solltest.

Das erste ist pass. Es tut buchstaeblich nichts - es ist ein Platzhalter fuer Stellen im Code, an denen Python eine Anweisung erwartet, du aber noch keinen echten Inhalt hast.

Das zweite ist break. Es bricht die innerste Schleife sofort ab, unabhaengig davon, ob die Schleifenbedingung noch erfuellt waere.

Das dritte ist continue. Es beendet den aktuellen Durchlauf vorzeitig und springt direkt zur naechsten Iteration - der Rest des Codeblocks wird in diesem Durchlauf uebersprungen.

Mit diesen drei Schluesselwoertern hast du volle Kontrolle ueber den Ablauf deiner Schleifen.
